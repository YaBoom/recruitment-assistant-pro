# 贡献指南

感谢您对本项目的关注！我们欢迎所有形式的贡献。

## 如何贡献

### 报告问题
- 使用 GitHub Issues 报告 bug
- 描述问题时请提供：
  - 问题描述
  - 复现步骤
  - 期望行为
  - 实际行为
  - 环境信息（操作系统、版本等）

### 提交功能请求
- 使用 GitHub Issues 提交新功能建议
- 描述功能的价值和使用场景

### 提交代码
1. Fork 本仓库
2. 创建您的特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交您的更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 打开一个 Pull Request

## 代码规范

### 提交信息规范
- `feat:` 新功能
- `fix:` 修复 bug
- `docs:` 文档更新
- `style:` 代码格式（不影响功能）
- `refactor:` 代码重构
- `test:` 测试相关
- `chore:` 构建过程或辅助工具的变动

### 代码风格
- 遵循项目现有代码风格
- 保持代码简洁清晰
- 添加必要的注释
- 编写单元测试
